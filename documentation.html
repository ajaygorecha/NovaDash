Complete guide to using NovaDash Admin Template

1. Installation

NovaDash is a ready-to-use admin template. No build tools or installation required. Simply:

  1. Download the template files
  2. Extract the ZIP file
  3. Open index.html in your browser

That's it! The template is fully functional with all features working out of the box.

3. Theme Customization
Changing Primary Color

To change the primary brand color, edit assets/css/theme.css:

:root {
  --primary: #ff0038; /* Change this value */
  --primary-hover: #e00030;
  --primary-light: rgba(255, 0, 56, 0.1);
}
Dark Mode

Dark mode is automatically enabled based on system preferences. You can also toggle it using the theme toggle button in the navbar. The preference is saved in localStorage.

4. Layout Options

NovaDash supports multiple layout options:

  • Default: Standard sidebar layout
  • Collapsed: Sidebar collapsed to icons only
  • Horizontal: Navigation moved to top
  • Boxed: Centered content with max-width

Switch layouts using the settings panel or by adding data attributes to the body element:

<body data-layout="sidebar-collapsed">
<body data-layout="horizontal">
<body data-layout="boxed">
5. Multi-language Support

NovaDash supports three languages: English, Hindi, and Arabic (RTL). Use the language selector in the navbar to switch languages.

Translation strings are defined in assets/js/i18n.js. To add more languages:

const translations = {
  en: { /* English */ },
  hi: { /* Hindi */ },
  ar: { /* Arabic */ },
  fr: { /* Add new language */ }
};
6. Using DataTables

DataTables is pre-configured for easy table enhancement. Simply add the dataTable class to any table:

<table class="table dataTable">
  <thead>...</thead>
  <tbody>...</tbody>
</table>

For more advanced usage, see assets/js/datatable-init.js.

7. Using FullCalendar

The calendar is initialized in assets/js/calendar-init.js. Add events using:

events: [
  {
    title: 'Meeting',
    start: '2024-01-15',
    backgroundColor: '#ff0038'
  }
]
8. Deployment on Vercel

The template includes vercel.json for easy deployment:

  1. Push your code to a Git repository
  2. Import the project to Vercel
  3. Deploy with default settings

The vercel.json file handles URL rewrites for SPA routing.
